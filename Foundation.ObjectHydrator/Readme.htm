Object Hydrator

Version 0.6
10/28/2009

Introduction:
Object Hydrator is a tool to ease the "Model First" development style and testing scenarios, by generating objects with pre-filled data.  Doing so allows you the freedom of working in your application with 'realistic' data at the same time giving you the flexibility to make changes to your model on the fly.

Using convention over configuration, Object Hydrator will pick an appropriate data type based on the type in the model and/or the name of the property and generate the data for you automatically.  With a fluent interface you can override these conventions easily and smoothly.

A very simple example is as follows:
Given this class

public class Customer
{
    public string FirstName {get;set;}
}

You can populate an instance of this class with a random real-world first name by first creating an instance of the Hydrator like so...

Hydrator<Customer> hydrator=new Hydrator<Customer>();

And then call the GetSingle method assigning it to a new customer object like this....

Customer mynewcustomer=hydrator.GetSingle();

You can also inject your own generators at instantiation time, as long as the implement the IGenerator interface. See Hydrator_InjectedGenerator_Test.cs

For more detailed examples please see the unit tests in Hydrator_SimpleCustomerTests.cs or the SampleWebApplication

Please see http://objecthydrator.codeplex.com for samples and documentation.
